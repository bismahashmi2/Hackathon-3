# Assessment Package: Module 14

## Components
- Capstone Proposal: 20% (Design document)
- Capstone Implementation: 40% (Working system)
- Capstone Demonstration: 25% (Live demo and presentation)
- Ethics Analysis: 15% (Ethical considerations documentation)

## Capstone Project Requirements

**Minimum requirements:**
- Integrate at least 3 course modules
- Include perception, planning, and control
- Demonstrate in simulation (hardware optional)
- Document design, implementation, and results
- Address ethical considerations

**Example projects:**
- Autonomous mobile manipulator for warehouse
- Assistive robot for elderly care
- Collaborative assembly system
- Search and rescue robot
- Agricultural harvesting robot

## Assessment Criteria

### Proposal (20%)
- Clear problem statement
- Well-justified design choices
- Realistic timeline and milestones
- Comprehensive ethics analysis

### Implementation (40%)
- Functional system meeting requirements
- Code quality and documentation
- Testing and validation
- Integration of multiple concepts

### Demonstration (25%)
- Successful live demonstration
- Clear presentation of results
- Analysis of performance
- Discussion of challenges and solutions

### Ethics (15%)
- Thoughtful analysis of ethical implications
- Concrete mitigation strategies
- Consideration of diverse stakeholders
- Long-term impact assessment

## Grading Rubric

| Component | Exceptional (90-100%) | Proficient (70-89%) | Developing (50-69%) | Beginning (<50%) |
|-----------|----------------------|---------------------|---------------------|------------------|
| Technical Merit | Novel integration, robust performance | Working system, meets requirements | Partial functionality | Non-functional |
| Documentation | Comprehensive, professional | Clear, adequate | Incomplete | Minimal |
| Presentation | Engaging, clear, insightful | Clear, covers key points | Unclear or incomplete | Poor |
| Ethics | Deep analysis, concrete proposals | Adequate consideration | Superficial | Absent |
